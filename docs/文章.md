前言
从未想过，当下知识的获取路径竟已如此便捷，成本竟已如此低廉。AI几乎直接把答案投送到跟前，学习开始变得前所未有地高效、直接、靶向。但也正因如此，一个问题也越来越明显：当我们省去了搜索、阅读、比较、怀疑，甚至是痛苦思考的过程，知识虽然来得更快了，可它在大脑中沉淀下来的厚度与深度，却不如传统的学习方式。过去为了弄懂一个问题，我们需要大量检索、反复阅读、来回求证，在这个过程中，认知被一点点撑开，记忆也因此变得更加牢固；而如今，快餐式的知识投喂形式，正在让“知道”变得越来越容易，也让“真正理解”变得越来越稀薄。

在脑机接口尚未到来、人类在技术意义上还未进化为“超级人类”之前，我更愿意把AI理解为一种能力的放大器。它像一根极长的杠杆，能放大人的效率、视野与产出；但杠杆能够撬动多大的世界，前提是你自身必须先有足够坚实的支点。若没有能力厚度与认知深度作为支撑，越便捷的知识获取方式，反而越可能击穿一个人的思考能力，让人停留在“仿佛懂了”的幻觉之中。

写这篇文章，主要有两个目的。其一，是逼自己真正静下心来，回到论文、代码与原理本身，认真阅读、理解，而不是停留在碎片化的信息消费和“看似懂了”的浅层兴奋里；其二，是亲手去做一件自己真正想做的事情——从零开始，构建一个22M参数的微型Transformer LLM，去靠近这个时代最核心的技术之一，也借此验证自己对它的理解究竟有多深。

当然，我本身并不是LLM领域的专家，文中难免会有理解不够准确甚至存在错误之处，欢迎交流与指正。

1、关于Transformer
我训练的模型并不大，参数量≈22M+。如果拿它去和千亿参数的LLM相比，几乎连人家零头都算不上。但在其结构上，它们遵循的是同一套架构范式，本质上没有区别，可谓是麻雀虽小五脏俱全。这个模型，我给它起名叫：Nova。

写这篇文章的目的，就是把Nova的原理、架构、源码彻底拆开并用工程化的陈述尽可能讲清楚，涉及到数学的地方，我也会尽量用大白话来翻译，至少确保不用在阅读的过程中还需要麻烦各位读者随时翻阅线性代数、微积分教材。在正式开始讲解Nova的架构之前，有一个前提绕不过去，那就是必须先把Transformer架构（本文基于Decoder-only Transformer）讲明白。否则后面在看到Embedding、Attention、FFN、Residual、LayerNorm等名词时一定蒙圈。

2017年，Google发表了一篇名为《Attention Is All You Need》的论文，直接改写了整个AI时代的走向，这篇论文首次提出了Transformer架构，随后便以极快的速度取代了当时在NLP领域长期占主导地位的RNN和LSTM架构，成为当今LLM的基石。而在这场架构革命的浪潮中，最早嗅到商机的是在游戏界以刀法闻名世界的老黄，他的远见在于，早在Transformer论文发表的前一年，就亲自向初创的OpenAI交付了世界上第一台专为AI设计的超级计算机DGX-1。当Transformer引发的大模型训练需求爆发式增长时，英伟达的GPU因其强大的并行计算能力，尤其是专为矩阵运算优化的Tensor Core和后续针对Transformer模型推出的Transformer Engine，迅速成为全球AI实验室和科技公司的“算力硬通货”。

回顾RNN模型，在模型的训练阶段，它的处理方式很像一条严格串行的流水线，必须先读第一个token，再读第二个，再读第三个，前一个时间步没算完，后一个时间步就无法开始。这种机制有两个非常致命的问题。首先是慢，由于计算天然串行，几乎无法充分利用GPU最擅长的大规模并行处理能力。你就算堆再多算力，很多时候也只能眼睁睁看着它按顺序一个个往前挪。其次是长距离依赖能力差。理论上RNN可以把前面的信息一路传到后面，但在实际训练中，句子一长，早期信息往往就已经在层层传递中衰减得差不多了。比如一句话：

```
“今天早上我和同事老汪一起在同程旅行APP上预定了一张从上海到北京的火车票。”
```

当模型终于读到”火车票”时，前面的”今天早上”在语义上其实仍然重要，只是在漫长的状态传递过程中，这部分信息往往已经衰减得非常微弱。Transformer 的处理方式则要直接得多：既然瓶颈出在”排队传递”本身，那就不再让信息排队。它让整个序列在一次前向传播中整体参与计算；在 Decoder-Only 架构下，每个位置只能回望自身及此前出现的上下文，不能提前看到未来的 token。这种因果约束只限制每个位置能访问哪些信息，却不妨碍所有位置在同一轮计算中并行完成注意力运算。而自注意力的本质，则是在这种因果掩码约束下，让每个位置都能从左侧历史 token 中按权重选择并聚合最相关的信息，形成当前的上下文表示。简单来说，访问有因果约束，计算却是并行的，这套机制就是 Self-Attention（自注意力）。于是，“今天早上”和”火车票”之间不再需要跨越漫长的中间状态才能关联，而是可以在注意力层里直接建立连接。它不仅让长距离依赖变得可达，也让序列计算得以摆脱 RNN 时代逐 token 串行推进的宿命。

为了把这件事讲得更具体，可以从三个层面来看：因果掩码决定谁能看见谁，并行计算决定为什么可以一起算，训练目标则决定为什么每个位置都必须参与计算。

以序列「今天 / 早上 / 买 / 了 / 机票」为例：
| 位置 | 可见范围 |
|------|----------|
| 位置 1（今天） | [今天] |
| 位置 2（早上） | [今天, 早上] |
| 位置 3（买） | [今天, 早上, 买] |
| 位置 4（了） | [今天, 早上, 买, 了] |
| 位置 5（机票） | [今天, 早上, 买, 了, 机票] |

先看因果掩码。假设输入序列是「今天 / 早上 / 买 / 了 / 机票」，那么在 Decoder-Only 模型里，每个位置的可见范围都是单向展开的：位置 1“今天”只能看到 [今天]；位置 2“早上”只能看到 [今天, 早上]；位置 3“买”只能看到 [今天, 早上, 买]；位置 4“了”只能看到 [今天, 早上, 买, 了]；位置 5“机票”只能看到 [今天, 早上, 买, 了, 机票]。这就是因果掩码施加的约束：每个位置只能访问自己以及左侧已经出现的 token，绝不能提前窥见右侧未来的信息。因此，“机票”确实可以直接关注“今天早上”，但它只能关注已经出现的“今天早上”，而不是未来尚未生成的内容。

```
┌────────┬────────┬────────┬────────┬────────┐
│ 位置1  │ 位置2  │ 位置3  │ 位置4  │ 位置5  │
│ 同时算 │ 同时算 │ 同时算 │ 同时算 │ 同时算 │
└────────┴────────┴────────┴────────┴────────┘
```

再看并行计算。很多人一看到“位置 5 能看到的东西比位置 3 多”，就会下意识觉得这几个位置必须一个接一个地算，仿佛又回到了 RNN 的接力模式。其实不是。Transformer 并不要求先算完位置 1，再把结果传给位置 2，再传给位置 3。它的做法是先把整段序列一次性表示成矩阵，然后一次性完成所有位置的 Q、K、V 投影，再一次性算出整张注意力分数矩阵，最后用因果掩码把那些“不该看的未来位置”整体屏蔽掉。于是，位置 1 到位置 5 虽然各自能访问的上下文范围不同，但它们的表示更新仍然可以在同一轮计算里一起完成。它们遵循的是同一套公式：QK^T → softmax → 加权求和；不同的不是计算框架，而是各自参与聚合的信息范围。就像几个人同时做同一种题型的阅读理解，解题方法一样，但每个人手里的题干长度不同，最终写出来的答案自然也不会相同。

最后是训练与推理。这也是很多人最容易混淆的地方。训练时，模型并不是只关心最后一个 token，而是让序列中的每一个位置都学会预测自己的下一个 token。以「今天 早上 买 了 机票」为例，训练时实际上是在同时做这样几件事：用“今天”预测“早上”，用“今天 早上”预测“买”，用“今天 早上 买”预测“了”，用“今天 早上 买 了”预测“机票”，再用“今天 早上 买 了 机票”预测下一个 token。换到对话场景里也是一样。比如「你好吗？」→「我很好」，训练时并不是只学习“你好吗？”→“我”，而是在同一轮前向传播里同时学习：“你”→“好”、“你 好”→“吗”、“你 好 吗”→“？”、“你 好 吗 ？”→“我”、“你 好 吗 ？ 我”→“很”、“你 好 吗 ？ 我 很”→“好”。所以，训练时关心的是所有位置，因为每个位置都是监督样本；推理时关心的通常只有最后一个位置，因为此时只需要生成下一个 token。

把这三层合在一起，你就能更准确地理解 Decoder-Only Transformer：因果掩码保证了自回归生成所需的单向信息约束，自注意力保证了远距离上下文可以直接建立联系，而矩阵化计算则让整段序列在训练阶段得以并行处理。 这也是为什么它既能保住“下一个 token 预测”这件事的因果正确性，又能在工程上彻底甩开 RNN 那种必须按时间步串行推进的旧范式。

如果把各种术语都暂时拿掉，Decoder-Only 模型做的事情，其实可以概括成三步：

```
第一步，把人类的文字变成计算机可以处理的数字表示。
第二步，把这些数字送进多层 Transformer Block 中反复加工，让模型逐步建立对上下文的理解。
第三步，在当前上下文的基础上，预测下一个最可能出现的 token，再把它接回输入里，继续重复这个过程。
```

所以，从本质上说，所谓“大模型会对话”，底层做的始终是同一件事：预测下一个 token。你给它输入“今天天气”，它会根据上下文判断，下一个 token 很可能是“很”；把这个结果接回去，输入变成“今天天气很”，它再继续预测下一个 token 可能是“好”；如此循环往复，一直到模型输出结束标记，整段回答才算真正完成。所以，LLM 看起来像是在“边想边打字”，其实它不是为了表现得更像人，而是因为它底层的生成机制，本来就是按 token 一步一步往前推出来的。

为了把 Transformer 讲透，我们不妨沿着一条真实的数据路径，完整追踪一次推理过程。

```
假设用户输入的是：“你好吗？”
模型希望生成的回答是：“我很好”
```

那么对于模型来说，它真正看到的输入，大致可以写成这样：
```
<s>你好吗？<sep>
             ↑ 从这里开始，模型要预测后续内容
```
接下来，这串输入会经历一条相当典型的流水线。第一站，是分词器（Tokenizer）。自然语言本身无法直接进入神经网络，所以首先要把文本切成 token，再把 token 映射成整数编号。于是“你好吗？”会变成类似这样的一串 ID：
```
[1, 42, 15, 88, 7, 3]
```
这里的 1 可能代表起始标记 <s>，3 可能代表分隔标记 <sep>。这些数字本身没有语义，它们只是符号在词表中的索引。

第二站，是嵌入层（Embedding）。整数 ID 依然不能直接承载语义，所以模型会把每个 token ID 映射成一个稠密向量。比如在 Nova 里，每个 token 会被映射成一个 384 维向量。你可以把它理解为：词表里每一个 token，都在模型内部对应着一根长度为 384 的数值坐标。与此同时，还要额外加入位置信息，告诉模型谁在前、谁在后，否则对它来说，“你好吗”和“好吗你”可能会变成同一堆词的无序集合。第三站，是多层 Transformer Block。这是最核心的计算区，在 Nova 中，输入向量会连续穿过 4 层 Transformer Block，而每一层内部又大致分为两步。

第一步是多头自注意力。在这里，序列中的每个 token 都会去评估：当前这句话里，哪些位置和我最相关？相关程度有多高？比如“吗”这个字，可能会发现自己和前面的“好”之间存在更强的关联，因为“好吗”在语义和语法上天然就是一个更紧密的局部结构。第二步是前馈网络（Feed-Forward Network）。如果说注意力负责“把相关信息找回来”，那么前馈网络更像一台局部加工机：它不改变序列之间的连接关系，而是对每个位置已经聚合到的信息进一步提炼和变换，让特征表达变得更抽象、更有判别力。

这一整套过程会重复多层。每多经过一层，模型对上下文的表征通常就会再抽象一层、再深入一层。前几层可能更多在处理局部模式和表面结构，后几层则逐渐形成更高层的语义组织。第四站，是输出层。经过所有 Transformer Block 之后，模型会取当前最后一个位置的隐藏状态，把它投影回整个词表空间，变成一个概率分布。这个分布回答的问题只有一个：“下一个 token 最有可能是谁？”

例如，模型可能给出这样的结果：
```
P(我) = 0.85
P(你) = 0.03
P(他) = 0.01
...
```
这时，“我”的概率最高，于是模型就输出“我”。然后，这个新生成的 token 会被拼接回原输入中，形成：
```
<s>你好吗？<sep>我
```

接着再走一遍同样的流程，模型继续预测下一个 token，得到“很”；再下一轮，得到“好”；直到某一轮预测出结束标记 <e>，整个回答才真正结束。所以，从外部看，模型像是在连续输出一句完整的话；但从内部看，它实际上只是在无数次重复同一个动作：基于当前上下文，预测下一个 token。

如果一定要把 Transformer 的优势压缩成最核心的两点，那么答案几乎都可以落回 Self-Attention 上。

第一，是全局视野。在自注意力机制下，序列中的每个位置都可以直接访问自身及此前已经出现的上下文，自主判断哪些历史信息与自己相关、相关程度有多高。它不再像 RNN 那样只能沿着时间步一点点传递状态，也不会因为距离太远就天然丢失联系。在 Transformer 的计算图里，句首和句尾之间不是隔着一条漫长走廊，而是始终保留着一条可以直接建立关系的路径。句子越长，这种优势越明显。

第二，是天然适合并行计算。由于整段序列可以整体送入矩阵运算，Transformer 几乎天生就是为 GPU 这类并行硬件准备的。相比之下，RNN 更像是单线程逐帧执行，哪怕后面站着再多算力，也很难真正展开。而 Transformer 的大部分核心操作，本质上都是大规模矩阵乘法，这正是现代加速硬件最擅长的战场。

也正是因为这两点，Transformer 才会成为今天几乎所有大模型系统的共同底座。从 2017 年到现在，无论是 GPT、Claude、LLaMA，背后都能看到 Transformer 的影子。形式可以变化，规模可以膨胀，工程实现也会一代代迭代，但那台真正推动一切的核心机器，始终还是这套机制。
