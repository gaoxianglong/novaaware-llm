训练精度（TF32 / AMP）与推理量化（4-bit / 5-bit）的区别
========================================================


一、训练时做的事情：针对的是"计算过程"，不碰权重本身
----------------------------------------------------

参考代码（train.py）：

    if torch.cuda.is_available():
        torch.set_float32_matmul_precision("medium")


### TF32 — 针对的是：矩阵乘法的"计算精度"

权重（比如 w_q.weight）始终存储为 FP32，32 位，4 字节，一个数字都没变。

TF32 做的事情是：当 GPU 执行矩阵乘法 weight × input 时，硬件内部把 FP32 的尾数
从 23 位截断到 10 位来加速计算，算完结果仍然输出为 FP32。

    权重存储:    FP32（不变）
    输入存储:    FP32（不变）
    计算过程:    GPU 内部偷偷用 TF32 精度做乘加（更快）
    输出结果:    FP32（不变）

权重从头到尾都是 FP32，TF32 只是 GPU 算矩阵乘法时的一个硬件加速策略。


### AMP — 针对的是：前向/反向传播中的"激活值和中间结果"

AMP 做的事情是：前向传播时，把某些层的输入和输出激活值临时转为 FP16 来计算
（比如 nn.Linear、nn.Conv2d），从而减少显存占用和加速计算。

但是：
  - 权重的 master copy 始终是 FP32
  - 梯度累加用 FP32
  - 优化器的状态（如 Adam 的动量）用 FP32
  - loss 计算、softmax、归一化这些数值敏感的操作保持 FP32

    权重存储:              FP32（始终不变，这是 master copy）
    前向传播某些层的激活值:  临时转为 FP16（算完就丢）
    反向传播的梯度:         FP32 累加
    优化器更新权重:         FP32
    保存 checkpoint:       FP32

所以训练结束后，保存的 best_model.pt 里的权重就是完整的 FP32。
TF32 和 AMP 从未修改过权重数据，它们只是让"计算过程"更快。


二、推理量化（4-bit / 5-bit）— 针对的是：权重本身
--------------------------------------------------

这是一个完全不同的东西。

量化做的事情是：训练完成后，把保存下来的 FP32 权重永久地压缩成更低位数的整数。

    训练完的权重:  FP32，每个参数 32 位（4 字节）
           ↓
        量化工具
           ↓
    量化后的权重:  INT4，每个参数 4 位（0.5 字节）← 体积变成 1/8

这里改的是权重本身，不是计算过程中的临时数据。

具体来说：模型里每个 nn.Linear 的 weight 矩阵，原来每个数字是一个 32 位浮点数
（比如 0.03125），量化后变成一个 4 位整数（比如 3），再配合一个共享的缩放因子
来还原近似值。

量化后推理时的过程是：

    存储:   INT4 整数权重 + scale 缩放因子
    推理时:  INT4 → 乘以 scale → 还原为近似的 FP16 浮点数 → 做矩阵乘法 → 输出


### 量化的数学原理举例（以 4-bit 对称量化为例）

假设 SwiGLUFFN 的 w1 的一行权重（384 个值中取 8 个示意）：

    原始 FP32 权重: [0.031, -0.187, 0.250, -0.062, 0.125, 0.312, -0.250, 0.003]

第 1 步：找到绝对值最大值
    abs_max = 0.312

第 2 步：计算缩放因子（scale）
    4-bit 有符号整数范围 -8 ~ +7
    scale = abs_max / 7 = 0.312 / 7 ≈ 0.0446

第 3 步：量化 —— 浮点数 → 整数
    0.031  / 0.0446 ≈  0.7 → 四舍五入 →  1
   -0.187 / 0.0446 ≈ -4.2 → 四舍五入 → -4
    0.250 / 0.0446 ≈  5.6 → 四舍五入 →  6
   -0.062 / 0.0446 ≈ -1.4 → 四舍五入 → -1
    0.125 / 0.0446 ≈  2.8 → 四舍五入 →  3
    0.312 / 0.0446 ≈  7.0 → 四舍五入 →  7
   -0.250 / 0.0446 ≈ -5.6 → 四舍五入 → -6
    0.003 / 0.0446 ≈  0.1 → 四舍五入 →  0

    存储: [1, -4, 6, -1, 3, 7, -6, 0]（每个 4 bit）+ scale = 0.0446

第 4 步：反量化 —— 推理时整数 → 浮点数
     1 × 0.0446 =  0.0446   （原值  0.031，误差 0.014）
    -4 × 0.0446 = -0.1784   （原值 -0.187，误差 0.009）
     6 × 0.0446 =  0.2676   （原值  0.250，误差 0.018）
    -1 × 0.0446 = -0.0446   （原值 -0.062，误差 0.017）
     3 × 0.0446 =  0.1338   （原值  0.125，误差 0.009）
     7 × 0.0446 =  0.3122   （原值  0.312，误差 0.000）
    -6 × 0.0446 = -0.2676   （原值 -0.250，误差 0.018）
     0 × 0.0446 =  0.0000   （原值  0.003，误差 0.003）

每个值都有量化误差，这些误差在矩阵乘法中会累积，最终影响 logits 输出。


### 4-bit vs 5-bit 的区别

    位数    整数范围          可表示的值    每个参数占用    量化误差
    ----    --------          ----------    ------------    --------
    FP32    ±3.4×10^38        ~40亿种       4 bytes         无（基准）
    FP16    ±65504            65536种       2 bytes         极小
    INT8    -128 ~ +127       256种         1 byte          小
    INT5    -16 ~ +15         32种          0.625 byte      中等
    INT4    -8 ~ +7           16种          0.5 byte        较大

5-bit 有 32 个档位，把同样的取值范围切成 32 份而不是 16 份，
四舍五入时的误差更小。


三、一句话总结区别
------------------

    项目               改了什么                    什么时候用      权重受影响吗
    ----               --------                    ----------      ------------
    TF32               矩阵乘法的硬件计算精度      训练时          不影响，权重始终 FP32
    AMP (FP16)         前向/反向传播中的临时        训练时          不影响，权重始终 FP32
                       激活值精度
    4-bit/5-bit 量化   权重参数本身的存储精度       训练后、推理时  直接改了权重，
                                                                   从浮点数压成整数

  - 前两个是训练加速手段，不碰权重
  - 后一个是模型压缩手段，直接改权重


四、量化对推理的影响
--------------------

速度/资源维度（正面影响）：
  - 模型体积缩小 6~8 倍
  - 显存占用大幅降低
  - 推理速度可能更快（内存带宽瓶颈减轻）

输出质量维度（负面影响）：
  - 大模型（7B+）：4-bit 量化后质量损失很小，日常使用几乎感觉不到
  - 小模型（如 Nova 的 1.3M）：量化后质量下降会比较明显，
    因为每个参数都在"承载信息"，压缩空间有限
  - 5-bit 比 4-bit 精度损失更小，是一个折中选择

模型越大，量化越"免费"；模型越小，量化越"昂贵"。


五、Nova 模型的情况
-------------------

Nova 模型约 1.3M 参数，FP32 下约 5MB，模型本身已经很小，
没有做 4-bit/5-bit 量化的必要。量化主要是给动辄几十 GB 的大模型使用的。
