推荐阅读顺序
第 1 站：config.py（129 行）
为什么先读它：它定义了整个项目用到的所有超参数（d_model、n_heads、n_layers、d_ff、max_seq_len……）。后面每个文件都会引用这些参数名，先读它可以建立全局参数"词汇表"，后续看到 config.d_model 直接知道是什么。

重点看：每个字段的 docstring，理解每个超参数的含义和取值原因。

第 2 站：tokenizer.py（408 行）
为什么第 2 读：它是数据进入模型的"大门"——文字 ↔ 数字 ID 的转换器。不理解分词器，后面看数据集和推理都会卡住。

阅读路线：

文件头的 ASCII 图（BPE vs 字符级对比）
特殊 token 定义（PAD/BOS/EOS/SEP/UNK，这 5 个贯穿全项目）
train_from_texts() — BPE 训练过程
encode() / decode() — 编码解码
save() / load() — 持久化

第 3 站：dataset.py（469 行）
为什么第 3 读：它把分词器产出的 token ID 包装成模型可以消费的 tensor（input_ids / target_ids）。这一步连接了"分词器"和"模型"。

阅读路线：

文件头的数据变换示例（原始 JSON → 拼接特殊标记 → 编码 → 截断/填充 → 构造 labels）
_encode_and_pad() — 理解 input_ids 和 target_ids 的关系（左移一位）
PretrainDataset — 预训练数据格式：<s> 纯文本 <e>
NovaDataset — 微调数据格式：<s> 问题 <sep> 回答 <e>
create_dataloader() — DataLoader 创建
load_pretrain_data() / load_qa_pairs() — 数据加载

第 4 站：model.py（933 行）— 核心，由内而外读
这是最重要的一个文件，建议按从小组件到大组件的顺序读，恰好就是文件中的排列顺序：

阅读顺序	类	行数	理解要点
4.1	RMSNorm	40-79	归一化：x / rms * gamma
4.2	SwiGLUFFN	85-203	前馈网络：展开→门控→压缩
4.3	MultiHeadAttention	209-422	自注意力：QKV→多头→因果掩码→输出
4.4	TransformerBlock	428-581	一层 Block：Norm→Attn→+x→Norm→FFN→+x
4.5	NovaModel	587-933	完整模型：Embedding→N层Block→Norm→Linear
核心理解目标：读完 4.4 时，你应该能画出一层 Block 的完整数据流；读完 4.5 时，能从 input_ids 追踪到 logits 的每一步。

第 5 站：train.py（848 行）
为什么第 5 读：有了模型和数据，才能理解训练怎么把它们串起来。整个训练流程的入口、初始化、核心循环、断点续训都在这里。

阅读路线（建议按以下顺序）：

5.1 main()（第 774-847 行）— 从入口开始，理解全局调度
  先读这里，了解整体流程：解析命令行参数 → 根据 --mode 调用不同的 setup 函数 → 调用 train() 执行训练。
  三种模式：pretrain（预训练）、finetune（微调）、不指定（向后兼容旧版）。

5.2 setup_pretrain()（第 423-531 行）— 预训练阶段初始化
  按 6 步顺序读：
    ① load_pretrain_data() — 加载 JSONL 纯文本语料
    ② NovaTokenizer.train_from_texts() — 训练 BPE 分词器（已有则跳过）
    ③ PretrainDataset + create_dataloader — 构建数据集
    ④ NovaModel().to(device) — 创建模型并移到 GPU/MPS/CPU
    ⑤ AdamW() — 创建优化器
    ⑥ load_checkpoint() — （可选）断点续训

5.3 setup_finetune()（第 548-662 行）— 微调阶段初始化
  流程和 setup_pretrain 类似，区别：
    - 用 load_qa_pairs() 加载 QA 问答数据（不是纯文本）
    - 直接加载预训练阶段保存的 BPE 分词器（不重新训练）
    - 用 NovaDataset（不是 PretrainDataset）
    - 加载预训练 checkpoint 时只要模型权重，不要优化器状态和 epoch（微调从 epoch 0 重新开始）
  重点理解：第 646-659 行的位置编码适配逻辑（预训练和微调的 max_seq_len 可能不同）

5.4 get_lr()（第 30-82 行）— 学习率调度器
  理解 Warmup + Cosine Decay 两阶段：
    阶段 1（Warmup）：学习率从 0 线性增长到 max_lr，避免训练初期参数更新过猛
    阶段 2（Cosine Decay）：学习率按余弦曲线从 max_lr 平滑下降到 min_lr，后期精细调参
  注意看文件中的 ASCII 图，一目了然。

5.5 train()（第 184-348 行）— 核心训练循环，最重要的函数
  逐块理解：
    ① 每个 epoch 开始：调用 get_lr() 设置本轮学习率（第 239-241 行）
    ② 遍历 dataloader 中的每个 batch（第 244 行）：
       - 取出 input_ids 和 target_ids（第 245-246 行）
       - 前向传播：model(input_ids) → logits（第 249 行）
       - 算 loss：cross_entropy(logits, target_ids)（第 250-254 行）
       - 反向传播：loss.backward()（第 257 行）
       - 梯度裁剪：防止梯度爆炸（第 259-261 行）
       - 优化器更新参数：optimizer.step()（第 262 行）
    ③ 每个 epoch 结束：打印日志、保存 checkpoint、记录 best_loss（第 295-330 行）
    ④ 训练结束：打印汇总信息（第 332-348 行）

5.6 load_checkpoint()（第 363-406 行）— 断点续训
  从 .pt 文件恢复 model 权重 + optimizer 状态 + epoch 编号，实现训练中断后继续。

5.7 辅助函数（第 88-173 行）— 日志格式化，快速浏览即可
  format_train_log() — 格式化每轮日志
  format_train_summary() — 格式化训练结束汇总
  should_log() — 判断本轮是否打印日志

第 6 站：chat.py（508 行）
为什么最后读：它是整个项目的"终点"——加载训练好的模型，执行推理。前面所有组件在这里汇合。

阅读路线：

load_model_for_inference() — 加载分词器 + checkpoint + 创建模型
generate() — 最值得细读：自回归生成的完整过程（构造 prompt → 循环前向传播 → 温度缩放 → top-k → 采样 → 追加 → 直到 <e>）
chat_loop() — 交互循环
main() — 命令行入口

```
阅读路径总览
config.py          超参数定义，建立全局"词汇表"
    │
    ▼
tokenizer.py       文字 ↔ 数字 ID，理解分词
    │
    ▼
dataset.py         原始数据 → tensor，理解 input_ids / target_ids
    │
    ▼
model.py           模型核心（由内而外）
 ├─ RMSNorm           归一化
 ├─ SwiGLUFFN         前馈网络
 ├─ MultiHeadAttention 自注意力
 ├─ TransformerBlock   一层 Block
 └─ NovaModel         完整模型
    │
    ▼
train.py           训练循环，把数据和模型串起来
    │
    ▼
chat.py            推理与对话，看最终效果
```
这个顺序的好处是：每读下一个文件时，它依赖的上游组件你都已经读过了，不会出现"这个东西从哪来的"的断裂感。
