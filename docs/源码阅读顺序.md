推荐阅读顺序
第 1 站：config.py（129 行）
为什么先读它：它定义了整个项目用到的所有超参数（d_model、n_heads、n_layers、d_ff、max_seq_len……）。后面每个文件都会引用这些参数名，先读它可以建立全局参数"词汇表"，后续看到 config.d_model 直接知道是什么。

重点看：每个字段的 docstring，理解每个超参数的含义和取值原因。

第 2 站：tokenizer.py（408 行）
为什么第 2 读：它是数据进入模型的"大门"——文字 ↔ 数字 ID 的转换器。不理解分词器，后面看数据集和推理都会卡住。

阅读路线：

文件头的 ASCII 图（BPE vs 字符级对比）
特殊 token 定义（PAD/BOS/EOS/SEP/UNK，这 5 个贯穿全项目）
train_from_texts() — BPE 训练过程
encode() / decode() — 编码解码
save() / load() — 持久化
第 3 站：dataset.py（469 行）
为什么第 3 读：它把分词器产出的 token ID 包装成模型可以消费的 tensor（input_ids / target_ids）。这一步连接了"分词器"和"模型"。

阅读路线：

文件头的数据变换示例（原始 JSON → 拼接特殊标记 → 编码 → 截断/填充 → 构造 labels）
_encode_and_pad() — 理解 input_ids 和 target_ids 的关系（左移一位）
PretrainDataset — 预训练数据格式：<s> 纯文本 <e>
NovaDataset — 微调数据格式：<s> 问题 <sep> 回答 <e>
create_dataloader() — DataLoader 创建
load_pretrain_data() / load_qa_pairs() — 数据加载
第 4 站：model.py（933 行）— 核心，由内而外读
这是最重要的一个文件，建议按从小组件到大组件的顺序读，恰好就是文件中的排列顺序：

阅读顺序	类	行数	理解要点
4.1	RMSNorm	40-79	归一化：x / rms * gamma
4.2	SwiGLUFFN	85-203	前馈网络：展开→门控→压缩
4.3	MultiHeadAttention	209-422	自注意力：QKV→多头→因果掩码→输出
4.4	TransformerBlock	428-581	一层 Block：Norm→Attn→+x→Norm→FFN→+x
4.5	NovaModel	587-933	完整模型：Embedding→N层Block→Norm→Linear
核心理解目标：读完 4.4 时，你应该能画出一层 Block 的完整数据流；读完 4.5 时，能从 input_ids 追踪到 logits 的每一步。

第 5 站：train.py（872 行）
为什么第 5 读：有了模型和数据，才能理解训练怎么把它们串起来。

阅读路线：

get_lr() — 学习率调度（Warmup + Cosine Decay）
train() — 核心训练循环（前向→算 loss→反向→梯度裁剪→更新）
setup_pretrain() — 预训练初始化流程
setup_finetune() — 微调初始化流程
load_checkpoint() — 断点续训
main() — 命令行入口，两阶段调度

第 6 站：chat.py（508 行）
为什么最后读：它是整个项目的"终点"——加载训练好的模型，执行推理。前面所有组件在这里汇合。

阅读路线：

load_model_for_inference() — 加载分词器 + checkpoint + 创建模型
generate() — 最值得细读：自回归生成的完整过程（构造 prompt → 循环前向传播 → 温度缩放 → top-k → 采样 → 追加 → 直到 <e>）
chat_loop() — 交互循环
main() — 命令行入口

```
阅读路径总览
config.py          超参数定义，建立全局"词汇表"
    │
    ▼
tokenizer.py       文字 ↔ 数字 ID，理解分词
    │
    ▼
dataset.py         原始数据 → tensor，理解 input_ids / target_ids
    │
    ▼
model.py           模型核心（由内而外）
 ├─ RMSNorm           归一化
 ├─ SwiGLUFFN         前馈网络
 ├─ MultiHeadAttention 自注意力
 ├─ TransformerBlock   一层 Block
 └─ NovaModel         完整模型
    │
    ▼
train.py           训练循环，把数据和模型串起来
    │
    ▼
chat.py            推理与对话，看最终效果
```
这个顺序的好处是：每读下一个文件时，它依赖的上游组件你都已经读过了，不会出现"这个东西从哪来的"的断裂感。
